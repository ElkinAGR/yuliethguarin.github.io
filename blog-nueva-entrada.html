Cómo utilizar Git: Tutorial Paso a Paso

Mi nombre es Elkin Giraldo yestoy en equipo con la compañera Yulieth en el trabajo colaborativo Git

Este tutorial te guiará a través de los pasos básicos para usar Git en tu proyecto. Aprenderás cómo inicializar un repositorio local, cómo crear ramas (branches) y cómo conectar tu repositorio local a un repositorio remoto en GitHub.

1. Inicialización del Repositorio Local

El primer paso para empezar a usar Git es inicializar un repositorio local en tu proyecto. Un repositorio es donde Git guarda todas las versiones de los archivos y su historial de cambios.

¿Cómo se hace?

En la terminal, navega a la carpeta de tu proyecto y usa el siguiente comando:

git init

Este comando crea una carpeta oculta llamada .git, donde Git almacenará toda la información relacionada con el control de versiones de tu proyecto.

Ejemplo Práctico:

Si tu proyecto está en una carpeta llamada "mi-proyecto", ve a esa carpeta y ejecuta el comando:

cd mi-proyecto
git init

Tu carpeta "mi-proyecto" ahora es un repositorio Git, y Git comenzará a rastrear los cambios que hagas en los archivos.

2. Creación de Branches (Ramas)

Las ramas te permiten trabajar en nuevas funcionalidades sin afectar el código principal (por lo general, la rama principal es main o master). Cada rama es una línea separada de desarrollo.

¿Cómo se crea una nueva rama?

Para crear una nueva rama, usa el siguiente comando:

git branch nombre-de-la-rama

Luego, para cambiarte a esa rama, usa:

git checkout nombre-de-la-rama

Ejemplo Práctico:

Imagina que estás trabajando en una nueva funcionalidad llamada "login". Crearías una nueva rama así:

git branch feature-login
git checkout feature-login

Ahora, cualquier cambio que hagas solo afectará a la rama feature-login, y no a la rama principal (main).

¿Cómo fusionar (merge) una rama con la rama principal?

Una vez que termines de trabajar en la rama, puedes fusionar los cambios con la rama principal:

git checkout main
git merge feature-login

Este comando integrará los cambios de feature-login en la rama main.

3. Conectar el Repositorio Local a un Repositorio Remoto en GitHub

Ahora que tienes tu repositorio local, es importante conectarlo a un repositorio remoto como GitHub, para poder colaborar con otros o simplemente tener una copia de seguridad en la nube.

Instrucciones:

Primero, crea un nuevo repositorio vacío en GitHub.

Luego, en tu terminal, conecta tu repositorio local a GitHub con el siguiente comando:

git remote add origin https://github.com/tu-usuario/tu-repositorio.git

Este comando asocia tu repositorio local con el repositorio remoto llamado origin.

Subir tus Cambios a GitHub:

Finalmente, para subir los cambios de tu repositorio local a GitHub, usa:

git push -u origin main

Este comando sube los cambios a la rama main en tu repositorio remoto en GitHub.

Ejemplo Práctico:

Imagina que tienes cambios locales que deseas subir a GitHub. Primero confirma tus cambios y luego empújalos al remoto:

git add .
git commit -m "Descripción de los cambios"
git push -u origin main

Con estos comandos, has confirmado tus cambios y los has subido a tu repositorio en GitHub.
